# PVP 特性层数设计

## 目标

在 PVP 对战伤害模拟的“增益状态”中新增“特性层数”控制，让用户手动表示战斗过程中已经累计的特性次数，并立即影响种族面板与伤害计算。

该功能只处理可累计特性。非累计、即时触发的特性继续沿用现有自动计算规则。

## 界面

- “特性层数”位于现有增益项目下方，单独占一行。
- 控件排列为：减号、整数层数、加号。
- 中间显示整数，不显示百分号。
- 减号最低只能减到 `0`。
- 加号没有最大层数限制。
- 当前精灵没有受支持的可累计特性时，不渲染该行。
- 行内显示当前实际特性名称，方便区分普通形态和首领形态。
- 特性层数产生的效果不在伤害明细中另行显示红字，避免与层数行重复表达。

## 状态模型

PVP 双方状态各新增：

```text
traitLayers: 非负整数
```

切换精灵时按新精灵的默认规则重置：

- 音速犬进化链：默认 `10` 层。
- 蹦床松鼠进化链：默认 `10` 层，对应出场时 10 能量。
- 其他支持层数的精灵：默认 `0` 层。
- 不支持层数的精灵：状态重置为 `0`，界面不显示控制行。

## 唯一计算来源

对于受支持的可累计特性，`traitLayers` 是累计效果的唯一来源：

- 不再同时执行旧的自动累计值。
- 不将旧值与层数值相加。
- 不在伤害明细中重复输出层数加成。
- 非累计特性不受此规则影响。

例如：

- 蹦床松鼠“囤积”10 层时，双防为 `+100%`，不能再额外叠加旧代码中的自动 `+100%`。
- 音速犬“专注力”10 层时，物攻为 `+100%`，不能再额外叠加旧代码中的首回合 `+100%`。

## 进化链与首领形态

普通进化链共用特性层数资格和每层规则。例如音速犬与护主犬必须采用同一规则。

首领形态不继承原特性名称，优先使用以下覆盖关系：

| 普通形态 | 首领或最高形态 | 特性变化 |
|---|---|---|
| 音速犬 | 风暴战犬 | 专注力 -> 全神贯注 |
| 黑猫巫师 | 黑猫密探 | 预警 -> 先知 |
| 恶魔狼 | 恶魔狼王 | 悲悯 -> 悼亡 |
| 彩蝶鲨 | 神谕鲨 | 水翼推进 -> 水翼飞升 |
| 梦想三三 | 奇梦咪 | 鼓气 -> 三鼓作气 |
| 棋绮后白子、棋绮后黑子 | 棋契陛下 | 渗透 -> 御驾亲征 |
| 女王蜂、花魁蜂后系 | 花魁蜂后 | 虫群鼓舞 -> 虫群突袭 |
| 火神 | 烈火战神 | 助燃 -> 爆燃 |
| 爵士鹿 | 波普鹿 | 蓄电池 -> 超级电池 |

覆盖表优先于当前远端数据源，防止远端数据仍把首领形态映射为普通特性。

规则表按实际特性名称定义每层效果，不按精灵名称复制计算代码。进化链只负责把精灵解析到相应规则。

## 计算接入

特性层数规则可以影响以下计算域：

- 物攻、魔攻、物防、魔防、速度百分比。
- 技能固定威力或技能威力百分比。
- 连击数。
- 技能能耗。
- 与能量相关的特性值。

计算顺序保持现有框架：

1. 根据精灵、形态和覆盖表确定实际特性。
2. 根据 `traitLayers` 生成特性修正。
3. 与性格、天分、技能增益、手动增益合并。
4. 计算最终六围。
5. 计算技能动态威力、连击数和最终伤害。

层数效果只应用一次。

## 蹦床松鼠能量规则

- 初始能量为 `10`，因此默认特性层数为 `10`。
- “囤积”每层提供物防 `+10%`、魔防 `+10%`。
- 使用技能扣除能量后，层数应同步为剩余能量。
- 当前版本即使尚未开放能量输入框，也保留层数作为当前能量的可手动表示。

## 音速犬规则

- 音速犬与护主犬进场默认 `10` 层。
- “专注力”每层提供物攻 `+10%`。
- 10 层等于物攻 `+100%`。
- 风暴战犬使用首领特性“全神贯注”，不得错误读取普通形态的“专注力”。

## 错误处理

- 不存在精灵、无对应规则或层数不是有效数字时，按 `0` 层处理。
- 任何减法都钳制到 `0`。
- 远端特性名称与首领覆盖表冲突时，以本地覆盖表为准。
- 切换到不支持层数的精灵时清除旧层数，避免残留影响伤害。

## 测试

至少覆盖：

1. 蹦床松鼠默认 10 层，双防只增加 100%，不重复叠加。
2. 蹦床松鼠从 10 减到 7 后，双防改为 70%。
3. 音速犬和护主犬默认 10 层，物攻增加 100%。
4. 普通精灵默认 0 层。
5. 减号不能低于 0，加号可以超过 10。
6. 普通进化链共享资格和规则。
7. 首领形态使用覆盖后的特性名称。
8. 切换精灵时层数按新精灵重置。
9. 特性层数效果参与最终伤害，但伤害明细不额外输出特性层数红字。
10. 不支持层数的特性继续执行原有即时规则。
