# 队伍配置完成反馈设计

## 目标

当某个队伍位置的常规配置全部填写后，让上方队伍概览卡片立即给出明确反馈，避免用户填写完下方表单后仍需自行确认是否完整。

## 完成条件

单个队伍位置必须同时满足以下条件才算配置完成：

1. 已选择精灵。
2. 已选择血脉。
3. 已选择性格。
4. 三个天分位置均已选择。
5. 四个技能位置均已选择。

`过山车目标技能` 是可选辅助配置，不参与完成状态判断。顶部 `使用过山车` 是否点击过也不影响完成状态或高亮动画。

## 概览卡片反馈

- 配置完成的队伍概览卡片持续显示独立的完成态高亮。
- 卡片内显示 `配置完成` 标记，使状态不只依赖颜色表达。
- 当前选中位置的蓝色边框与完成态同时存在，不让完成态覆盖当前编辑位置提示。
- 任一必填项被清除后，卡片立即移除完成态高亮和 `配置完成` 标记。

## 动画

- 当某个位置由未完成首次变为完成时，播放一次短暂高亮动画。
- 普通切换队伍位置、重新渲染页面、点击 `使用过山车` 或修改可选的过山车目标技能，不重复播放动画。
- 完成后清除任一必填项，再次补齐时可再次播放动画。
- 动画遵循 `prefers-reduced-motion`，用户选择减少动态效果时不播放。

## 状态与渲染

- 使用一个纯完成判断函数读取标准化队伍位置数据，不依赖 DOM 样式判断。
- 概览卡片渲染根据完成判断函数添加完成态类和标记。
- 维护当前已完成位置集合，只把本次从未完成变为完成的位置标记为待播放动画。
- 精灵或技能选择会重建队伍编辑器；其他下拉选择目前只保存数据，因此这些更新路径需要统一刷新对应概览卡片状态。
- 动画状态仅用于界面反馈，不写入 `localStorage`，页面重新加载时显示持续完成态但不补播动画。

## 验证

新增回归测试验证：

1. 完成判断要求精灵、血脉、性格、三个天分和四个技能全部存在。
2. `rollerSkillId` 是否存在不影响完成判断。
3. 完成卡片包含完成态类和 `配置完成` 标记。
4. 从未完成变为完成时添加一次动画类。
5. 清除必填项后完成态消失，再次补齐可重新触发动画。
6. 过山车目标技能变化和顶部过山车操作不触发完成动画。
7. 减少动态效果媒体查询禁用完成动画。
8. 全部静态测试、内联脚本解析和浏览器交互验证通过。
