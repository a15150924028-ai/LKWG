# style.md — 微信小程序 PVP 对战工具 Liquid Glass UI 设计规范

## 1. 设计方向

产品是微信小程序 PVP 对战工具，核心页面包括：

- 队伍配置页
- 阵容分析页
- PVP 伤害模拟页
- 精灵选择弹窗
- 技能选择弹窗
- 隐藏数据维护页

整体风格采用：

> Liquid Glass / 液态玻璃 / 浅色半透明卡片 / 柔和渐变 / 圆润按钮 / 轻游戏 HUD

不要做成传统表单工具，也不要做成深色赛博风。  
目标是让当前复杂的配置页面变得更像一个“高级小程序战术面板”。

---

## 2. 当前 UI 问题

现有页面功能完整，但视觉上存在这些问题：

1. 表单密度过高，页面显得拥挤。
2. 卡片之间层级不够明显。
3. 输入框、标签、技能说明、数值块风格不统一。
4. PVP 页面字段太多，用户第一眼压力较大。
5. 阵容分析页数值很多，但视觉重点不够突出。
6. 底部 Tab 比较普通，缺少产品识别度。
7. 当前页面偏“后台表单”，需要升级成“轻游戏工具”。

---

## 3. 视觉关键词

关键词：

- Liquid Glass
- 毛玻璃
- 半透明卡片
- 浅色渐变背景
- 蓝紫青绿色点缀
- 圆角胶囊按钮
- 柔和阴影
- 清爽留白
- 数据卡片化
- 移动端优先

整体感觉：

> 像一个干净、轻盈、有科技感的战斗计算器，而不是复杂的数据录入表。

## 4. 全局配色

### 4.1 页面背景

页面背景使用中性浅色柔和渐变，不使用纯白；允许极淡的淡蓝、淡紫背景光感，但不能形成大面积实体色块。

```css
--bg-page: linear-gradient(180deg, #F8FAFC 0%, #F4F7FB 100%);
--bg-glow-blue: rgba(77, 163, 255, 0.12);
--bg-glow-purple: rgba(108, 99, 255, 0.10);
--bg-glow-cyan: rgba(70, 216, 207, 0.07);
```

背景光斑只能作为低不透明度的氛围层：

- 允许淡蓝、淡紫、淡青绿的低透明度模糊光斑。
- 光斑必须靠页面边缘或卡片边缘，不能压在正文下面。
- 光斑透明度建议 0.07 - 0.12，不要超过 0.14。
- 不要在 hero、说明卡、特性卡、配置卡内部铺大面积彩色底。

不要使用复杂纹理，不要影响文字可读性。

### 4.2 主色

```css
--primary: #6C63FF;
--primary-light: #8B7CFF;
--secondary: #46D8CF;
--secondary-light: #8DEFE8;
--accent-blue: #4DA3FF;
--accent-green: #35D49B;
--accent-red: #FF6B7A;
--accent-orange: #FFB45C;
```

主按钮优先使用蓝紫渐变。  
辅助按钮使用青绿色渐变。  
危险操作使用浅红色背景，不要用大面积纯红。

### 4.3 文字颜色

```css
--text-main: #1F2633;
--text-sub: #5F6B7A;
--text-muted: #8A96A8;
--text-light: #FFFFFF;
--text-danger: #E85A6A;
--text-success: #1FAE7A;
```

文字不要太黑，主文字使用深蓝灰，不使用纯黑。

### 4.4 卡片颜色

```css
--card-glass: rgba(255, 255, 255, 0.52);
--card-glass-strong: rgba(255, 255, 255, 0.66);
--card-border: rgba(255, 255, 255, 0.85);
--card-border-blue: rgba(77, 163, 255, 0.42);
--card-border-green: rgba(53, 212, 155, 0.42);
```

卡片要有液态玻璃感：

```css
background: rgba(255,255,255,0.52);
backdrop-filter: blur(24px);
border: 1px solid rgba(255,255,255,0.85);
box-shadow:
  0 16px 40px rgba(80, 110, 160, 0.16),
  inset 0 1px 0 rgba(255,255,255,0.90),
  inset 0 -1px 0 rgba(255,255,255,0.36);
```

## 5. 字体规范

使用微信小程序系统字体，不额外引入字体。

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
```

字号：

```css
--font-page-title: 22px;
--font-section-title: 17px;
--font-card-title: 15px;
--font-body: 14px;
--font-small: 12px;
--font-mini: 11px;
--font-number-large: 28px;
--font-number-medium: 18px;
```

字重：

```css
--weight-title: 700;
--weight-section: 600;
--weight-body: 400;
--weight-number: 700;
```

数字可以适当加粗，让伤害值、种族值、属性值更醒目。

---

## 6. 间距规范

以 8px 为基础单位。

```css
--space-page-x: 14px;
--space-card: 14px;
--space-section: 16px;
--space-small: 8px;
--space-mini: 6px;
```

页面规则：

- 页面左右边距：14px 或 16px
- 卡片之间：12px - 16px
- 卡片内边距：14px - 16px
- 表单项之间：8px - 10px
- 大模块之间：18px - 22px

不要让卡片互相贴太近。  
当前 UI 的问题是信息太密，需要通过留白降低压力。

---

## 7. 圆角规范

Liquid Glass 风格必须保持大圆角。

```css
--radius-page-card: 24px;
--radius-card: 20px;
--radius-input: 14px;
--radius-button: 999px;
--radius-chip: 999px;
--radius-small: 10px;
```

使用规则：

- 大卡片：20px - 24px
- 输入框：14px
- 标签：999px 胶囊
- 主按钮：999px 胶囊
- 底部 Tab：顶部大圆角或胶囊高亮

---

## 8. 阴影规范

阴影要柔和，不要厚重。

```css
--shadow-card: 0 10px 28px rgba(80, 110, 160, 0.14);
--shadow-float: 0 14px 36px rgba(90, 110, 180, 0.20);
--shadow-button: 0 8px 20px rgba(108, 99, 255, 0.28);
--shadow-green: 0 8px 20px rgba(70, 216, 207, 0.24);
```

按钮、Tab 选中态、重要结果卡可以有轻微发光。

# 9. 全局组件

## 9.1 顶部导航栏

当前微信小程序顶部已有默认胶囊区域，不要抢占太多空间。

页面顶部结构：

```text
页面标题
副标题 / 数据版本 / 当前状态
```

示例：

```text
我的队伍
配置六只精灵的血脉、性格、天分和技能
```

样式：

- 页面标题 22px，加粗
- 副标题 13px，灰蓝色
- 外层用玻璃卡片包住
- 卡片高度约 86px - 96px
- 圆角 24px

## 9.2 底部 Tab Bar

底部 Tab 保留 3 个：

```text
队伍
分析
PVP
```

样式：

- 背景：半透明白色毛玻璃
- 高度：64px - 72px
- 顶部边框：半透明白线
- 选中项：蓝紫渐变文字或小胶囊背景
- 未选中项：灰蓝文字

选中态：

```css
background: linear-gradient(135deg, rgba(108,99,255,0.15), rgba(70,216,207,0.15));
color: #4DA3FF;
border-radius: 999px;
```

不要只改变文字颜色，最好有一个浅色玻璃胶囊底。

## 9.3 主按钮

主按钮用于：

- 使用过山车
- 立即计算伤害
- 保存队伍
- 一键套用配置

样式：

```css
height: 42px;
border-radius: 999px;
background: linear-gradient(135deg, #7C6DFF 0%, #4DA3FF 100%);
color: #FFFFFF;
font-weight: 600;
box-shadow: 0 8px 20px rgba(108, 99, 255, 0.28);
```

按钮文字居中。  
按钮左右内边距 18px - 24px。

## 9.4 次按钮

用于：

- 撤回过山车
- 清空
- 重新选择
- 展开详情

样式：

```css
background: rgba(255,255,255,0.58);
border: 1px solid rgba(255,255,255,0.85);
color: #5F6B7A;
border-radius: 999px;
```

危险按钮：

```css
background: rgba(255, 107, 122, 0.10);
color: #E85A6A;
```

## 9.5 输入框 / 选择器

当前页面很多选择框，要统一风格。

样式：

```css
height: 40px;
background: rgba(255,255,255,0.68);
border: 1px solid rgba(190, 205, 225, 0.55);
border-radius: 14px;
padding: 0 12px;
color: #1F2633;
```

选中 / 聚焦状态：

```css
border-color: rgba(77, 163, 255, 0.65);
box-shadow: 0 0 0 3px rgba(77, 163, 255, 0.12);
```

右侧清除按钮 `x` 使用灰色，不要太抢眼。

## 9.6 标签 Chip

用于属性、定位、血脉、天气、克制关系。

基础样式：

```css
display: inline-flex;
align-items: center;
height: 22px;
padding: 0 8px;
border-radius: 999px;
font-size: 11px;
font-weight: 500;
```

属性标签可以使用浅色底：

```css
火：rgba(255,107,122,0.13)
水：rgba(77,163,255,0.13)
草：rgba(53,212,155,0.13)
电：rgba(255,209,102,0.18)
机械：rgba(120,140,170,0.15)
翼：rgba(140,180,255,0.16)
```

# 13. 页面具体视觉稿描述

## 队伍页视觉稿

页面背景是浅蓝白渐变，上方有一个大玻璃标题卡。  
标题卡写“我的队伍”，下面小字写“配置六只精灵的血脉、性格、天分和技能”。

# 14. 禁止项

不要出现以下设计：

1. 不要使用纯白背景。
2. 不要使用大面积深黑背景。
3. 不要使用密集表格。
4. 不要让输入框边框过重。
5. 不要让页面像后台管理系统。
6. 不要使用第三方游戏图片。
7. 不要外链图片。
8. 不要保留 BWiki、rocomwiki、patchwiki、biligame 等来源站痕迹。
9. 不要在普通用户页面显示导入、导出、清除数据。
10. 不要把高级参数全部默认展开。
11. 不要使用淡蓝、淡紫、淡青绿色的大面积实体背景色块；如果使用光斑，必须低透明度、柔和、靠边，并且不影响文字可读性。

# 15. 实现优先级

1. 全局背景改为浅色 Liquid Glass 渐变。
